<objective>
Establish the entire greenfield app shell and the local end-to-end pin+URL slice: a Vite + React 19 + TypeScript (pinned 6.0.3) scaffold, a single-pin Leaflet map centered on the Czech Republic (D-05) with CARTO voyager tiles (D-03) and default attribution (D-04), click-to-place + drag-to-adjust pin interaction (D-06, LOC-01), and lat/lng/zoom encoded to the URL query string as the single source of truth (D-09, D-10, LOC-03). The map never auto-recenters after a pin is placed (D-07).

Purpose: This is the Walking Skeleton's structural backbone plus its persistence-equivalent boundary. Per PROJECT.md the app has no backend and no database; the URL query string IS the persistence layer (write on pin move, read on load). This plan proves that boundary works locally before geocoding (Plan 02) and deployment (Plan 03) layer on top.

Output: A runnable local app where a user can load the map, click to drop a pin, drag to adjust, watch the URL update, and reload to reproduce the exact view — with a tested pure URL-state module underneath.
</objective>
