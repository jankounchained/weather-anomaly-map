## Layout & Interaction Notes

*(Not a template-required section, but load-bearing context for planner/executor ‚Äî this phase establishes the app shell.)*

- **Map-dominant layout.** The Leaflet map fills the primary/left viewport region ‚Äî effectively the full window minus the docked panel.
- **Fixed-width info panel** docked to the right edge, full viewport height, width 360px on desktop. This is the app's single non-map UI surface in Phase 1, and the location it will grow in: Phase 2 stacks temperature + anomaly stats below the location display in this same panel; Phase 3 stacks the 7-day chart below that. Build the panel container now with this future stacking in mind (vertical flex, top-to-bottom sections), even though Phase 1 only fills the first section.
- **No pin exists on initial load.** Map centers on the Czech Republic default (D-05) with no marker placed. The panel shows the Empty State (see Copywriting Contract) until the user's first click.
- **First click places the pin immediately** (D-06); the panel transitions Empty ‚Üí Loading ‚Üí Resolved (place name, or coordinate fallback per D-02) without the map moving (D-07 ‚Äî no auto-recenter/zoom, ever, in this phase).
- **Panel background uses the Secondary (30%) color** to visually separate it from the map area; the map/page background is Dominant (60%).
- **Leaflet's default chrome stays default:** zoom control (top-left) and attribution control (bottom-right, D-04) are not restyled or repositioned. Do not let the docked panel overlap or cover either control.
- **Desktop-only for this phase.** Mobile-responsive layout is explicitly deferred to v2 (PLAT-03) ‚Äî no breakpoints or touch-target sizing need to be designed now.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 400 | 1.4 |
| Heading | 20px | 600 | 1.2 |

Usage: **Display** is reserved for an optional app title at the top of `LocationPanel` (implementation discretion ‚Äî include if it reads well, omit if the panel feels cramped; not required by any Phase 1 requirement). **Heading** renders the resolved place name. **Body** renders the empty-state and loading-state copy. **Label** renders the coordinate fallback line (`lat, lng`) and any secondary/small text.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Page background; visible map-container background before tiles paint |
| Secondary (30%) | `#F4F5F7` | `LocationPanel` background ‚Äî visually separates panel from map |
| Accent (10%) | `#2563EB` | Pin marker, loading indicator, keyboard focus outlines ‚Äî see reserved-for list below |
| Destructive | `#DC2626` | Reserved for future destructive actions ‚Äî **not used anywhere in Phase 1** (no delete/reset/destructive action exists in this phase's scope) |

Accent reserved for: **(1)** the map pin/marker's icon color (if a custom marker replaces Leaflet's default pin, it uses this color; if Leaflet's default blue marker icon is kept as-is, that satisfies this token without custom styling), **(2)** the loading-state indicator/spinner shown during the ~3s geocoding timeout, **(3)** the `:focus-visible` outline on the map container and on the `LocationPanel` (keyboard accessibility). Accent is never used for body text, panel background, or Leaflet's tile imagery.
