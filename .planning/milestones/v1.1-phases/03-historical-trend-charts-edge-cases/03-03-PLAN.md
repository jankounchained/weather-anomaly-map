## Phase Goal (user story)

**As a** person checking a location's weather, **I want to** see each of the last 7 days plotted against that day's own 30-year historical distribution — with every year's values visible and the average emphasized — plus a clear message when a day lacks enough history, **so that** I can tell at a glance how unusual the whole recent week has been, not just today.

> Plan 03 is the visual payoff slice: it turns Plan 02's data + Plan 01's recharts into the actual on-screen "week strip". After this plan, a real user pins a location and sees, below today's anomaly headline, seven mini distribution charts — the exact capability VIZ-01/VIZ-02 promise — with graceful "Not enough data" placeholders (ROBU-01) where history is thin. It runs in Wave 2 because it needs both recharts installed (Plan 01) and the data pipeline + per-day computation (Plan 02).

<objective>
Render the 7-day trend row (VIZ-01, VIZ-02, ROBU-01): a horizontal small-multiples strip below the AnomalyCard where each of the last 7 days shows its own 30-year historical distribution as translucent jittered dots, a bright emphasized mean line, and a distinct actual-value diamond — all on one shared y-scale — with a bordered "Not enough data" placeholder for any day lacking usable history.

Purpose: This is the phase's user-facing deliverable. Task 2 builds and smoke-tests a single mini-chart first (the 03-RESEARCH.md Assumption A1 spike: confirm recharts 3.9.2's Scatter/ReferenceLine props render dots + line under jsdom before scaling to 7), then Task 3 composes the row and wires it into App with the correct gating.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Open-Meteo-derived values (dates, temperatures, computed jitter/domain) → React/Recharts DOM | API-derived data is rendered into SVG chart marks and labels/tooltips |
| Per-day series (samples/actual/mean) → recharts internals | Non-finite or mismatched values could produce NaN y-positions or a chart-layer crash |
