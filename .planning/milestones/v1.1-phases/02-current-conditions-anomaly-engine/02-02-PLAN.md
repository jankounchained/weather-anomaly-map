## Phase Goal (recap)

**As a** visitor checking a location, **I want to** see today's temperature and how it compares to the 30-year norm for this calendar day, **so that** I can tell at a glance whether today is unusually warm or cold.

> Plan 02 completes the goal's differentiator: the anomaly. It extends Plan 01's `AnomalyCard` from "current temperature only" to "current temperature + delta hero + verdict headline + z-score badge + data-quality tooltip", backed by the 30-year baseline and the pure anomaly math module.

<objective>
Deliver the anomaly-engine vertical slice (ANOM-01, ANOM-02, ANOM-03, ANOM-04): for a pinned location the user sees an accurate, easy-to-interpret comparison of today's temperature against a statistically sound 30-year day-of-year baseline — a whole-number °C delta (hero), a plain-language verdict (headline), and a z-score (secondary badge) — all revealed together after both data calls resolve.

Purpose: This is the core value of the whole product ("the anomaly score must be accurate and easy to interpret at a glance"). The math is a pure, dependency-free, unit-tested module in the `src/lib/coords.ts` mold; the baseline fetch reuses the `useReverseGeocode` hook idiom; the render extends the Plan 01 `AnomalyCard`.

Output: `src/anomaly/{types,anomaly,anomaly.test}.ts`, `src/weather/useHistoricalBaseline.ts`, plus `getHistoricalBaseline` added to `src/weather/client.ts`, and the extended `AnomalyCard` / `App` / `App.css`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser client → Open-Meteo archive API | Untrusted upstream JSON (30-year series) crosses here; empty/partial series must not reach the math module |
| Baseline series → anomaly.ts (pure math) | Degenerate inputs (zero variance, <2 samples) could produce NaN/Infinity |
| Anomaly-derived text (delta/z-score/verdict) → React DOM | API-derived values rendered into the UI |
