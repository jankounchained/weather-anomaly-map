## Color

Extends the existing `--color-chart-*` token family in `src/index.css`. **Two new tokens pairs (fill + stroke) for the violin halves; every other mark keeps its existing token verbatim** — this is a locked decision (PD-12: "The mean ticks and actual diamond keep their existing tokens"), not a re-derivation.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (`--color-dominant`) | Unchanged — map/page background |
| Secondary (30%) | `#f4f5f7` (`--color-secondary`) | Unchanged |
| Accent (10%) | `#2563eb` (`--color-accent`) | Unchanged usage list, **plus** now applies to TWO per-half mean ticks per tile (see below) instead of one full-width line — same token, doubled application |
| Destructive | `#dc2626` (`--color-destructive`) | Unchanged — no destructive actions this phase |

**New chart tokens (add to `src/index.css` alongside the existing `--color-chart-*` block):**

| Token | Value | Usage |
|-------|-------|-------|
| `--color-chart-prior-fill` | `rgba(87, 83, 78, 0.24)` | Prior-25yr half violin body fill. Same base hue as `--color-chart-historical` (continuity: "prior" = the app's existing neutral historical-grey), alpha nudged from 0.22→0.24 since it now fills a solid curve area instead of sparse dots. |
| `--color-chart-prior-stroke` | `#57534e` (= `--color-anomaly-normal`) | Prior-25yr curve outline, opaque. Reuses the app's existing "near-normal" anomaly-scale grey. |
| `--color-chart-recent-fill` | `rgba(154, 52, 18, 0.26)` | Recent-5yr half violin body fill. Reuses `--color-anomaly-hot`'s rust hue at low alpha — the "warmer, the one to watch" half per PD-12, drawn from the app's own anomaly-color vocabulary rather than an unrelated new hue. |
| `--color-chart-recent-stroke` | `#9a3412` (= `--color-anomaly-hot`) | Recent-5yr curve outline, opaque. |

**Marks that keep their EXISTING token (locked, not re-decided):**

| Mark | Token | Value |
|------|-------|-------|
| Both per-half mean ticks (recent AND prior) | `--color-chart-mean` | `var(--color-accent)` = `#2563eb` — **same token both sides**, distinguished by tile position (left half vs right half), not by hue |
| Actual-value diamond | `--color-chart-actual` | `#ea580c`, 1px white stroke — unchanged |
| Rug-fallback dots (either half, when it degrades) | `--color-chart-historical` | `rgba(87, 83, 78, 0.22)` — **unchanged, and deliberately NOT tinted** with the new prior/recent hues. A rug always reads as "raw/uncertain" the same way it always has, visually distinct from a confident curve regardless of which side degraded. |

Accent reserved for: **InfoTooltip trigger glyph/border, InfoTooltip trigger focus ring, the loading-spinner's spinning arc, and the trend chart's per-half mean ticks** (carried over from Phase 6, extended to cover the now-doubled mean-tick usage). Never applied to the violin body fill/stroke, panel headlines, or body copy.

**Color rationale (why reuse anomaly-scale hues instead of new ones):** the app already has a semantic warm/cool vocabulary (`--color-anomaly-cold` #1e3a8a → `--color-anomaly-normal` #57534e → `--color-anomaly-hot` #9a3412) driving the Delta panel's continuous color lerp. Deriving prior=grey/recent=rust from that existing scale keeps the new violin colors legible as "the same app's color language," and keeps recent-fill (`#9a3412` family) visually distinct from the actual-value diamond (`#ea580c`) — darker/rustier vs. brighter/more saturated orange — so the two marks don't get confused even though both sit in the warm family.

## Violin Geometry & Marks Contract

**Shared vertical scale (unchanged mechanism, extended input set):** one shared padded y-domain (D-06) computed once and passed to every tile + the axis column, exactly as today. `computeSharedYDomain`'s input set changes from `[...day.samples, day.actual, day.mean]` to `[...recentSamples, ...priorSamples, actual, recentMean, priorMean]` across all 7 days — same padding behavior (~8%), just flattening both halves instead of one combined sample array.

**Per-day build (`buildViolinPaths`):**
1. Compute both halves' KDE curves using **one shared pooled Silverman bandwidth** per day (`silvermanBandwidth([...priorSamples, ...recentSamples])`, factor=1.0) — per the spike finding, NOT per-half bandwidth (biases the recent half artificially flatter).
2. `sharedMax = max(peak(recentCurve), peak(priorCurve))` — PD-06's shared density peak; both halves scale against this ONE number so relative peakiness stays comparable within a day.
3. **Curve grid is clamped to each half's own `[sampleMin, sampleMax]`**, not the full padded y-domain — prevents a curve's tail from visually touching or crossing the tile frame (the spike's explicit "don't let the tail kiss the border" guidance), while still rendering on the one shared y-scale.
4. Half path: `x = cx + side · (density / sharedMax) · maxHalfWidth`, `side = −1` for prior (left), `+1` for recent (right); path closes back to `cx` at top and bottom of the curve's clamped range.
5. Per half, render either:
   - `n ≥ 20` (`halfDrawsCurve`, spike-pinned `n_min`) → filled curve path, 1.5px stroke, in that half's fill/stroke tokens.
   - `n < 20` → rug: sample dots (3px radius, existing `historicalDotShape` styling, `--color-chart-historical` token) jittered horizontally via the existing `jitterX` helper, **bounded within that half's own `[cx, cx ± maxHalfWidth]` band** — jitter must never cross the center line into the other half's territory.

**Retained marks (unchanged behavior, new per-half doubling where noted):**
- **Two mean ticks** (PD-07): short horizontal line at `y(mean)`, extending `side · 29px` from `cx`, in `--color-chart-mean` (both ticks, same token). The vertical gap between the two ticks is the climate-shift read — no additional annotation needed this phase (TREND-04 stays deferred).
- **Actual-value diamond** (PD-08): unchanged `makeActualShape`, rendered at `x = cx` (the center line, not offset to either half) — reused verbatim, same 5px size, same `#ea580c` fill / white 1px stroke.
- **Shared Y-axis column** (PD-09): `TrendYAxisColumn` unchanged, rendered once to the left of the 7-tile row.

**Whole-tile placeholder (PD-03, unchanged):** the existing bordered "Not enough data" placeholder (verbatim copy, verbatim styling) is reserved for a true zero-sample day only, governed by the existing `hasUsableSampleCount` gate — unaffected by whether an individual half falls below the new `n_min=20` curve threshold.

**Accessibility (native SVG, no raw-HTML sink — T-03-07):**
- Each half's mark group (curve OR rug) wraps a native `<title>` — see Copywriting Contract below for exact strings. This gives screen-reader users the same "why does this side look different" context sighted users get from the curve/rug visual contrast.
- Individual rug dots keep NO per-dot tooltip (existing "dots represent the distribution as a whole, not individually-labeled points" policy — unchanged from `historicalDotShape`'s current behavior).
- Both mean ticks and the actual diamond keep native `<title>` tooltips (mean ticks are new this phase; diamond is unchanged).

## UI Considerations

Applicable state considerations resolved: 7 covered, 1 backstop, 0 unresolved.

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| populated (happy path) | Per-day tile, both halves ≥ n_min | ✅ covered | Dual-curve render is the default path — spike-verified real per-half sizes are always ≈55 (recent) / ≈275 (prior), both well above `n_min=20`; ERA5 has no data deserts. Geometry fully specified above. |
| populated / partial | Per-day tile, ONE half < n_min | ✅ covered | Thin half degrades to rug (PD-01); healthy half still draws its curve. Geometry, tokens, and accessible title specified above. | populated / partial | Per-day tile, BOTH halves < n_min | ✅ covered | Dual rug (PD-02); shared Y-axis still comparable across days. Same rug styling both sides — reads as "no curve today," not an error. |

| overflow | Violin curve vs. tile frame | ✅ covered | Curve grid clamped to each half's own sample min/max (not the full padded domain) plus an 8px edge gutter (`maxHalfWidth`) — tail never touches the tile border. See Violin Geometry & Marks Contract. |
| long-text | Mean-tick / diamond / rug native `<title>` tooltips | ✅ covered | All tooltip strings are short, bounded, single-line native SVG `<title>` text with fixed format patterns (no dynamic long strings, no user-authored content this phase). |

| populated | Legend (`TrendLegend`) | 🧪 backstop | Draft copy provided above (5 items) is functionally complete, but PD-10 explicitly requires a reviewer copy round-trip before the wording is final — exact strings are NOT locked by this UI-SPEC. At verify time, treat legend copy as needing explicit reviewer sign-off evidence, not a silent pass. |
