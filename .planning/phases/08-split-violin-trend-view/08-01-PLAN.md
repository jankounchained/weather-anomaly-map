<objective>
Build the pure statistics + data-shape core the whole split-violin tile stands
on, bottom-up: a hand-rolled Gaussian KDE + Silverman module (`kde.ts`), the
per-half curve-vs-rug gate (`halfDrawsCurve`, n_min=20), and the transformation
of `computeTrendDay` from a single-sample result into a two-sample
(recent-5yr / prior-25yr) result — re-windowing the SAME already-fetched
`baseline.daily` series with no new fetch.

Purpose: every downstream plan (geometry, render, legend) consumes these
exports. Getting the math and the discriminated-union shape right first
isolates the phase's highest-risk numerical work from the render rewrite.
Output: `src/anomaly/kde.ts` (+ tests), a two-sample `TrendDayResult`, and a
two-sample `computeTrendDay`.
</objective>

<assumption_delta_decision>
Signal: pluralization — `computeTrendDay`'s return transitions from a single
sample result (`samples` / `mean`) to a two-sample result (`recentSamples` /
`priorSamples` + `recentMean` / `priorMean`); the `TrendDayResult` `usable:true`
member becomes the two-half model.
Now-primary noun: the two-half (recent / prior) trend-day model.
Decision: promote — the two-sample shape becomes the primary and ONLY
`usable:true` `TrendDayResult` shape; the old single-sample shape is retired, NOT
kept alongside.
Rationale: every consumer (`computeSharedYDomain`, `buildViolinPaths`,
`TrendDayChart`) migrates to the two-half model in this same phase; a
compatibility shim would be dead weight and would risk two shapes drifting apart.
Invariant test: a contract test asserts the `usable:true` union member exposes
`recentSamples` / `priorSamples` / `recentMean` / `priorMean` / `actual` and no
longer exposes `samples` / `mean`.
</assumption_delta_decision>
