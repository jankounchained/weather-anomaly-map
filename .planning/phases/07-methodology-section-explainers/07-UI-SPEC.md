## Color

Unchanged from `src/index.css` (already declared, 60/30/10 split in production):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (`--color-dominant`) | Map background, page background |
| Secondary (30%) | `#f4f5f7` (`--color-secondary`) | Docked panel background (empty state), z-score chip pill background |
| Accent (10%) | `#2563eb` (`--color-accent`) | InfoTooltip trigger border/glyph/focus ring (Phase 6), loading-spinner active arc, chart mean reference line, **+ Methodology `<summary>` `:focus-visible` ring (new, same purpose class: keyboard-focus indication, not a new usage category)** |
| Destructive | `#dc2626` (`--color-destructive`) | Error-state copy only — unchanged, no new destructive use this phase |

Accent reserved for: **InfoTooltip trigger glyph/border + focus ring, the loading-spinner's spinning arc, the trend chart's mean reference line, and the Methodology disclosure `<summary>`'s `:focus-visible` ring.** Never applied to panel headlines, body copy, the Δ number, the percentile line, or the disclosure chevron itself.

**Percentile line color:** inherits the page's default body text color (no accent, no `var(--anomaly-color)`) — it restates the same fact as the color-driven Δ number in plain language, so it must not compete visually with that number's Phase 5 color-driven focal treatment (LAYOUT-03 dominance rule, carried over unchanged).

**Disclosure chevron color:** `text-muted` (matches `PanelHeadline`'s existing label color) — decorative/structural, not an interactive-accent element, so it does not draw from the accent reservation.

**No new tokens.** No color-scale legend is introduced (REQUIREMENTS.md "Out of Scope").

## Copywriting Contract

No primary CTA exists in this phase — Phase 7 is an information/explanation addition, not an action flow.

| Element | Copy |
|---------|------|
| Primary CTA | N/A — no new action/button introduced this phase |
| Methodology panel — `<summary>` headline (always visible, PD-11) | "How This Works" |
| Methodology panel — subsection 1 header | "What This Shows" |
| Methodology panel — subsection 1 body | "This tool shows how unusual today's temperature is at any location, compared to what's typical for that exact calendar day over the past 30 years." |
| Methodology panel — subsection 2 header | "How It's Computed" |
| Methodology panel — subsection 2 body (pipeline, plain-language, PD-12) | "We pull a 30-year historical baseline for this spot from Open-Meteo's archive — temperatures within 5 days of today's date, every year. We average those readings to get a 'typical' value for this date. Today's reading minus that average gives the delta (Δ) in °C. The z-score expresses that same gap in standard deviations, so it stays comparable across places with different day-to-day variability. The percentile shows where today ranks among those historical years for this date." |
| Methodology panel — trust/limitation notes (PD-13, appended after subsection 2, same flat level — no 3rd heading) | "Current and historical data comes from Open-Meteo, using ERA5 reanalysis for the 30-year baseline — modeled climate data, not a single weather station, which is why the delta is shown in whole degrees rather than decimals. Some remote locations (oceans, deserts) don't have enough historical coverage, so the anomaly and percentile may be unavailable there." |
| Percentile line — warmer case (p > 55%, PD-07) | "Warmer than {p}% of years for this date." |
| Percentile line — colder case (p < 45%, PD-07) | "Colder than {100−p}% of years for this date." |
| Percentile line — near-median case (45% ≤ p ≤ 55%, PD-07) | "Around the middle for this date." |
| Percentile line — suppressed (PD-04, `zScore === null`) | No line rendered — omit entirely, do not render an empty/placeholder row. |
| Empty state heading | N/A — Methodology panel has no empty state (PD-11: always visible, static copy, not selection-gated); percentile line has no standalone empty state (it lives inside DeltaPanel's existing Empty/Loading/Error branches — see Phase 6 UI-SPEC, unchanged this phase) |
| Empty state body | N/A — see above |
| Error state | N/A — no new error surface; percentile suppression on null variance (above) is the only "no value" case, and it is silent-omission by design (PD-04), not an error message |
| Destructive confirmation | N/A — no destructive actions in this phase |

**Numeric formatting rule:** the percentile renders as a whole-number integer + `%` (no decimal place), matching the existing whole-number-delta precision convention (`formatDelta`, Pitfall 4) — avoids implying station-level precision from what is empirically ranked modeled/reanalysis data.

**Median-band threshold (Claude's discretion, locks PD-07's "right around the middle" for implementation):** 45–55% (inclusive) reads as "Around the middle for this date"; above 55% uses the warmer framing; below 45% uses the colder framing. Symmetric ±5-point band around the p=50 median, chosen for a clean, easily-testable rule with no additional token/config needed.

## Component Inventory (new/extended this phase)

| Component | Responsibility | Style source |
|-----------|-----------------|---------------|
| `MethodologyPanel` (new, `src/app/MethodologyPanel.tsx` — lean toward a new file per one-component-per-file convention, per CONTEXT.md discretion note) | Renders a native `<details>` inside `PanelShell`, with `<summary>` carrying `PanelHeadline`'s exact classes + a trailing chevron glyph. Always mounted (PD-11), regardless of `hasSelection`/anomaly-resolution state — a sibling of `LocationDisplay`/`TrendRow` inside `LocationPanel`'s children, after `TrendRow`. | `PanelShell` + `PanelHeadline` verbatim (Phase 6); no new visual primitive |
| Percentile line (extends `DeltaPanel.tsx`'s populated branch) | Inserted between the verdict `<p>` and the z-score chip `<p>` (PD-06). Plain `<p className="m-0 text-body font-body">` — no new wrapper, no chip. | Matches the existing micro-copy `<p>` styling already used above the verdict in the same file |

**Disclosure visual/interaction spec (PD-10):**
- `<details>` renders as a plain block inside `PanelShell` (no extra border/background beyond the shell itself — one glass card, not a nested card).
- `<summary>` classes: `PanelHeadline`'s exact string (`text-label leading-[1.5] font-semibold text-muted uppercase tracking-[0.05em]`) + `flex items-center justify-between gap-sm cursor-pointer list-none [&::-webkit-details-marker]:hidden focus-visible:outline focus-visible:outline-2 focus-visible:outline-accent` — suppresses the native marker (avoids a double chevron) and adds the accent focus ring (WCAG keyboard-visibility parity with `InfoTooltip`'s trigger).
- Chevron: a `▸` glyph in a `<span aria-hidden="true">`, `text-muted`, rotated via `details[open] & { transform: rotate(90deg) }` (or an equivalent `group-open:` Tailwind pattern) — **gated `motion-safe:transition-transform duration-200`** so reduced-motion users get an instant, non-animated rotation (mirrors the existing `motion-safe:` prefix convention in `DeltaPanel`/`.panel-backdrop`).
- **No expand/collapse height animation.** The native `<details>` open/close toggles instantly (no CSS grid-rows/height trick). This is a deliberate simplicity choice: a height-animated native `<details>` requires either JS-measured height or the still-unreliable `::details-content`/`interpolate-size` CSS, which is disproportionate engineering for a single flat disclosure. Because there is no height motion to gate, the `prefers-reduced-motion` requirement (PD-10, ROADMAP research flag) is trivially satisfied by the *absence* of animation — verify only the chevron's own rotation transition disables under the OS reduced-motion toggle (there is nothing else to check).
- Body content wrapper: `<div className="flex flex-col gap-sm pt-sm">` holding the two subsections + trailing trust notes, each as plain `<p>`/`<div>` elements — no nested `<details>`, no third heading level (single flat disclosure, per REQUIREMENTS.md "Out of Scope: nested/multi-level disclosure").

## UI Considerations

Applicable state considerations resolved: 8 covered, 0 backstop, 0 unresolved.

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| populated | Percentile line (Delta panel) | ✅ covered | Three copy branches defined and bounded — warmer (>55%), colder (<45%), near-median (45–55%) — see Copywriting Contract. |
| null / degenerate-variance | Percentile line (Delta panel) | ✅ covered | Per PD-04, when `anomaly.zScore === null` the line is omitted entirely (no placeholder text, no "N/A" row) — silent suppression is the specified behavior, matching the existing z-chip's own null-variance handling in the same panel. |
| overflow, long-text | Percentile line (Delta panel) | ✅ covered | Single short static sentence with a 1-3 digit dynamic number, inside the existing 760px `PanelShell` column — same bounded-content profile as the panel's other micro-copy (Phase 6 precedent). |
| always-visible (PD-11) | Methodology panel | ✅ covered | Mounts unconditionally as a `LocationPanel` child in every state (idle/empty/loading/error/resolved) — no `hasSelection` or `isAnomalyReady` gate, confirmed against `LocationPanel.tsx`'s always-rendered children slot. |
| long-text | Methodology panel body | ✅ covered | All copy is static/authored (no dynamic per-location text, per REQUIREMENTS.md "Out of Scope: dynamic methodology copy") and wraps normally inside the 760px column — no truncation risk. |
| overflow | Methodology panel (expanded) | ✅ covered | Panel sits inside `LocationPanel`'s `overflow-y-auto` scrollable aside; expanding adds page-flow height, never clips or requires its own internal scroll region. |
| interaction (keyboard + motion) | Methodology disclosure toggle | ✅ covered | Native `<details>`/`<summary>` is keyboard-operable (Enter/Space) and mouse-operable with zero custom JS; motion requirement satisfied by having no height animation to gate, only a `motion-safe:`-gated chevron rotation (see Component Inventory). |
| consistency-risk (accepted tradeoff) | Percentile line vs. z-score verdict tier | ✅ covered | Empirical percentile rank and the parametric z-score verdict tier are computed from the same samples but by different methods (PD-01 rationale: distribution-free, honest on skewed climates) — they can rarely disagree at the edges (e.g. "much warmer" verdict alongside a percentile in the high-80s on a skewed distribution). This is an intentional, already-decided tradeoff (not a bug for planner/executor/checker to "fix"), documented here so it isn't re-litigated downstream. |
