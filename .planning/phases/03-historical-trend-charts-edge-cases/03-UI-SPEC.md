> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-rolled components styled with plain CSS + CSS custom properties, same convention as Phase 1/2 (`src/index.css` tokens, BEM-style class names) |

Plain CSS + custom-property tokens remain sufficient.

## Layout & Interaction Notes

*(Not a template-required section, but load-bearing context for planner/executor.)*

### Panel width (resolves 03-RESEARCH.md Open Question 1 / Pitfall 4; updated 03-05 gap closure)

- **`location-panel` widens from 360px to 760px** (both the `flex: 0 0 360px` and `width: 360px` declarations in `src/app/App.css` change to `760px`). Original locked decision (03-01..03-04) sized the panel to 720px assuming the leftmost tile alone carried the Y-axis tick labels inside its own 88px chart width. The 03-05 gap-closure fix moves the Y-axis out of the tile row entirely into its own shared column (see "Chart Visual Encoding" → "Y-axis (temperature)" below), which needs its own explicit row width budget: a 40px axis column + `var(--space-sm)` (8px) gap to the chart row + 7 mini-charts at 88px + 6× 8px inter-tile gaps = 712px of row content width. 760px panel minus 32px left/right padding (`--space-md` × 2, unchanged) leaves 728px of content width — enough for the row with 16px of breathing room, no horizontal scroll needed.
- Widening (not horizontally scrolling) was chosen per 03-RESEARCH.md's own recommendation: it matches D-06's "compare at a glance" intent, where every day's chart is simultaneously visible.
- `map-region` keeps `flex: 1 1 auto` — it absorbs the width difference automatically; no change needed there. This assumes a desktop viewport ≥1280px (consistent with the app's existing no-breakpoints, desktop-first stance — mobile is deferred to v2 PLAT-03).
- `location-panel`'s existing `overflow-y: auto` is untouched — vertical scroll (if the panel's total content height exceeds viewport height) continues to work exactly as before; no new horizontal overflow is introduced.

### Trend row placement and hierarchy (D-07)

- The new trend-row block is the **third** stacked section inside `location-panel__content` (after `LocationDisplay`, after `AnomalyCard`), using the panel's existing `gap: var(--space-md)` (16px) flex spacing — no CSS change needed to `location-panel__content` itself.
- Visual hierarchy is established through typography weight/size, not extra spacing: the trend row's own section heading ("Last 7 Days") renders as a small uppercase eyebrow label — deliberately smaller/quieter than `AnomalyCard`'s `--font-size-display` delta and `--font-size-heading` verdict — so the eye lands on today's headline anomaly first, per D-07.
- Internal spacing within the trend-row block: `var(--space-sm)` (8px) between the eyebrow heading and the 7-chart row itself.
- The trend row has **no outer card/border chrome** — it sits directly on the panel's existing `--color-secondary` background, consistent with `AnomalyCard`'s own borderless style. No new "card" visual language is introduced at the row-container level (only the per-day placeholder slots get a border — see Chart Visual Encoding).

### Gating: when does the trend row render at all?

- Mirrors `AnomalyCard`'s existing combined-gate pattern (D-09 from Phase 2): the trend row renders **nothing** (not even placeholders) until `current.status === 'resolved' && baseline.status === 'resolved'` — same two hooks `AnomalyCard` already gates on, since D-13 extends `useCurrentWeather` rather than adding a third hook.
- If the combined fetch resolves but yields no usable `recentDaily` series at all (e.g. malformed/missing `daily` field per the V5 guard in 03-RESEARCH.md's Security Domain), the trend row **omits itself entirely** rather than rendering 7 placeholder cards — a total data outage is already communicated once, by `AnomalyCard`'s own error state above it. Do not duplicate an error message below.
- Once `recentDaily` has valid dates+values, D-09's per-day usable-sample threshold is evaluated independently for each of the 7 days (D-11) — each slot renders its own chart or its own placeholder (D-12/D-14), never both, never omitted.

### Component Inventory

| Component | Responsibility |
|-----------|-----------------|
| `TrendRow` | Composes the eyebrow heading, a shared `TrendYAxisColumn`, 7× `TrendDayChart`, and the persistent `TrendLegend`; computes the shared padded y-`domain` once (Pattern 3, 03-RESEARCH.md) and passes it identically to the axis column and every chart child. Renders nothing if the combined current+baseline gate isn't resolved-with-data (see above). |
| `TrendDayChart` | Renders exactly one of two states for a single day, at a fixed 88×140px footprint: **(a)** populated — `ComposedChart` with translucent historical `Scatter`, distinct actual-value `Scatter` (diamond), and bright `ReferenceLine` (mean), own `YAxis` always hidden; **(b)** placeholder — bordered empty-state card with short copy, same footprint, per D-12/D-14 (one visual path regardless of cause) |
| `TrendYAxisColumn` (new, 03-05 gap closure) | Renders the row's single shared, visible `YAxis` once, in its own 40px explicit-width column to the left of the 7-tile row, on the identical shared `domain` — the ONLY place a temperature tick label renders (VIZ-02 Gap 1 fix) |
| `TrendLegend` (new, 03-05 gap closure) | Persistent, always-visible key explaining the three chart marks (pale dot / bright line / orange diamond) via native SVG swatches reusing the exact `--color-chart-*` tokens the marks render with — closes VIZ-02 Gap 2 (no hover required) |

No form inputs, buttons, or modals are added in this phase — the trend row is entirely read-only, consistent with Phase 1/2's interaction model (map click/drag remains the only user action in the app).

## Chart Visual Encoding

*(Not a template-required section — this is the concrete mark spec D-01 through D-06 need to be actionable for the planner/executor, translating "pale" / "bright" / "distinct" into exact values.)*

### Per-slot footprint (applies to both populated and placeholder states)

| Property | Value |
|----------|-------|
| Slot width | 88px (fixed — no `ResponsiveContainer`, per 03-RESEARCH.md Pitfall 3). Now identical across all 7 slots — every tile's own `YAxis` is hidden (see "Y-axis (temperature)" below), so all 7 plot areas are pixel-equal (03-05 gap closure, VIZ-02 Gap 1). |
| Slot total height | 140px |
| Weekday/"Today" label row height | 16px |
| Gap between label row and chart/placeholder body | `var(--space-xs)` (4px) |
| Chart/placeholder body height | 120px |
| Gap between the 7 slots | `var(--space-sm)` (8px) |

### Shared Y-axis column (new, 03-05 gap closure — VIZ-02 Gap 1)

Rendered once, to the LEFT of the 7-tile `.trend-row__charts` row, inside a `.trend-row__axis` column that mirrors each tile's own label-row + chart-row layout so its ticks line up vertically with every tile's plot area:

| Property | Value |
|----------|-------|
| Column width | 40px (fixed, explicit — `AXIS_WIDTH` in `TrendDayChart.tsx`, reused by both the standalone axis and any per-tile `YAxis` that might ever be shown) |
| Spacer height (aligns with the label row above every tile) | 16px, empty (no text) |
| Gap between spacer and axis chart body | `var(--space-xs)` (4px), matching every tile |
| Axis chart body height | 120px — identical to `CHART_HEIGHT`, so ticks line up with every tile's plot area |
| Gap between the axis column and the 7-tile chart row | `var(--space-sm)` (8px) |

### Weekday / "Today" label (renders above every slot, populated or placeholder)

| State | Copy | Size | Weight | Color |
|-------|------|------|--------|-------|
| Prior 6 days | 3-letter weekday abbreviation (e.g. `Mon`, `Tue`, locale `en-US` `weekday: 'short'`) | `--font-size-label` (14px) | `--font-weight-label` (400) | `#4b5563` (muted — same value already used by `AnomalyCard`'s `__zscore` chip) |
| Rightmost slot (today, D-08) | `Today` | `--font-size-label` (14px) | 600 (semibold — the project's other declared weight) | `--color-accent` (`#2563EB`) |

The "Today" slot is visually identifiable by label alone (bold + accent color), independent of its position — reinforces D-08 without relying on spatial memory only.

### Populated chart marks (D-01 through D-04, D-06)

| Mark | Recharts primitive | Fill / Stroke | Notes |
|------|---------------------|----------------|-------|
| Historical dots (~30 years, D-01) | `Scatter` | fill `rgba(37, 99, 235, 0.22)` (translucent `--color-accent`), no stroke | Radius ~3px. Jittered x-position only (D-04) — precompute via a deterministic seeded offset in a `useMemo`, never `Math.random()` inline in render (03-RESEARCH.md Anti-Pattern) |
| Mean reference line (D-03) | `ReferenceLine` | stroke `--color-accent` (`#2563EB`), `strokeWidth: 2`, solid | `ifOverflow="visible"` — **not** the Recharts default `"discard"` — so a mean sitting at the shared domain's edge stays visible rather than silently vanishing (03-RESEARCH.md Pitfall 2). This is a locked visual requirement, not optional polish. |
| Actual-value marker (D-02) | `Scatter` (single point) | fill `#EA580C` (new token, see below), `1px` white stroke, `diamond` shape | Deliberately a second hue (orange vs. the blue family used for dots/mean/accent) so it reads as visually distinct at a glance per D-02, not just a differently-shaped dot in the same color family. Reserved *exclusively* for this marker — never used elsewhere in the UI, same reserved-for discipline as `--color-destructive`. |
| Y-axis (temperature) | `YAxis` | tick color `#4b5563`, `--font-size-label` (14px), explicit `width` — never the Recharts default ~60px reservation | **(Updated 03-05 gap closure, VIZ-02 Gap 1.)** Every one of the 7 `TrendDayChart` tiles now hides its own `YAxis` (`showYAxis={false}` for all 7, no per-index asymmetry) — the axis renders exactly ONCE, via a standalone `TrendYAxisColumn`, in its own 40px-wide explicit-width column to the LEFT of the 7-tile row. This makes all 7 tiles pixel-identical (same plot-area width for the dot cloud / mean line / diamond), fixing the previous defect where the leftmost tile's inline visible ticks ate into its own fixed chart width and looked squished relative to its 6 siblings. `TrendYAxisColumn` uses the identical shared `domain` prop (D-06) as every tile, so the axis stays perfectly aligned to the shared scale. |
| X-axis | `XAxis` | — | `type="number"`, fixed `domain={[0, 1]}`, `hide` — x carries no semantic meaning (D-04); never render x ticks/labels/gridlines |
| Domain padding | — | — | Computed once in `TrendRow` per 03-RESEARCH.md Pattern 3 (10% padding, floor/ceil) — generous enough that every day's mean/dots/actual fits without triggering the `ReferenceLine` overflow case above |
| Tooltip / hover detail | native `title` attribute on the actual-value marker only | — | e.g. `"Jul 12 — 23°C (+2° vs. 30-yr avg)"`. No Recharts `Tooltip` component in v1 — matches the codebase's existing `title`-attribute pattern (`AnomalyCard`'s info button) rather than introducing a new interaction affordance. Historical dots get no individual tooltip — they represent the *distribution*, not individually-labeled data points. |
| Legend (new, 03-05 gap closure — VIZ-02 Gap 2) | `TrendLegend` — native SVG (`circle`/`rect`/`polygon`), not Recharts | fills reuse `--color-chart-historical` / `--color-chart-mean` / `--color-chart-actual` verbatim | Persistent, always-visible key rendered once below the 7-tile row (not per-tile, not hover-only) so a first-time viewer never has to hover the actual-marker tooltip to learn what the marks mean. Three entries: pale dot swatch = "Each of the last 30 years", bright line swatch = "30-year average", orange diamond swatch = "That day's temperature" — see Copywriting Contract for the exact strings. |

**New color tokens this phase adds to `src/index.css`:**
```css
--color-chart-historical: rgba(37, 99, 235, 0.22); /* translucent accent, D-01 */
--color-chart-mean: var(--color-accent);            /* bright, D-03 — reuses accent, no new hue */
--color-chart-actual: #ea580c;                       /* distinct marker, D-02 — new hue, reserved for this use only */
--color-muted: #4b5563;                              /* formalizes the value already used inline in AnomalyCard's zscore chip */
--color-border-subtle: #e5e7eb;                       /* new — placeholder card border only */
```

### Placeholder state (D-09 through D-12, D-14 — one visual path for both causes)

| Property | Value |
|----------|-------|
| Background | `--color-dominant` (`#FFFFFF`) — intentionally lighter than the panel's `--color-secondary` background, so the placeholder reads as a distinct "card" against the panel |
| Border | `1px solid var(--color-border-subtle)` (`#E5E7EB`), `border-radius: 8px` |
| Text | centered, `--font-size-label` (14px), `--font-weight-label` (400), color `--color-muted` (`#4b5563`) |
| Visible copy | `Not enough data` (fits 2 short lines at 88px width) |
| Accessible/full copy | `title="Not enough history for this day"` attribute + matching `aria-label` on the placeholder's root element — the full D-12 message is always available to assistive tech and on hover, even though the visible glyph text is shortened to fit the fixed slot width |
| Icon | none — text-only, no dash/glyph decoration |

This exact placeholder renders for **both** a genuine data desert (D-09's stricter threshold failing) and a fetch-timing edge case (D-14) — there is only one placeholder component variant; the cause is never surfaced to the user.

## Spacing Scale

Declared values (must be multiples of 4) — unchanged from Phase 1/2, reused as-is:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between a slot's weekday label and its chart/placeholder body (new this phase) |
| sm | 8px | Gap between the 7 trend-row slots (new this phase); gap between the "Last 7 Days" eyebrow heading and the row (new this phase) |
| md | 16px | `location-panel` internal horizontal padding (unchanged); gap between stacked panel sections including the new `TrendRow` block |
| lg | 24px | `location-panel` top/bottom edge padding (unchanged) |
| xl | 32px | Reserved, not used this phase |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions: the 88×140px `TrendDayChart` slot footprint and the 720px panel width are **component/layout sizing decisions, not spacing-token values** — they don't need to be multiples of the 4px spacing scale (they aren't gaps/padding), though 88, 140, and 720 all happen to already be multiples of 4.

## Typography

Declared values — unchanged from Phase 1/2, reused as-is (no new sizes/weights introduced this phase):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 400 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Usage this phase: **Label** (14px/400) renders weekday abbreviations, y-axis tick labels, and placeholder card text. **Label at 600 weight** (borrowing the project's other declared weight, not a new one) renders the "Today" slot's caption and the "Last 7 Days" eyebrow heading, both in `--color-accent`/`--color-muted` respectively rather than a larger size — hierarchy is carried by weight/color, not by introducing a 5th font size.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Page/map background (unchanged); placeholder card background (new this phase) |
| Secondary (30%) | `#F4F5F7` | `location-panel` background (unchanged, now at 720px width) |
| Accent (10%) | `#2563EB` | Pin marker, loading spinners, focus outlines (unchanged) + **new this phase:** mean reference line (`--color-chart-mean`), historical dots at 22% opacity (`--color-chart-historical`), "Today" slot caption |
| Destructive | `#DC2626` | Reserved for future destructive actions — still unused anywhere in the app, including this phase (no destructive action exists in Phase 3's scope) |
| Chart — actual marker (new, phase-scoped) | `#EA580C` | The single actual-value diamond in each populated `TrendDayChart` — exclusively this use, never a button/link/other UI element |

Accent reserved for: **(1)** the map pin/marker, **(2)** loading-state spinners (`LocationDisplay`, `AnomalyCard`), **(3)** `:focus-visible` outlines, **(4)** `AnomalyCard`'s info-button border/text, **(5, new)** the trend chart's mean reference line and translucent historical dots, **(6, new)** the "Today" slot's caption text. Accent is never used for body text, panel background, or the actual-value marker (which deliberately uses the new orange token instead, to stay visually distinct from the accent-colored dots/line it's plotted against).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | No CTA exists in this phase (trend row is read-only, matches Phase 1/2's map-click-only interaction model — no new buttons/links added) |

| Actual-marker hover detail (native `title`, not visible body copy) | `"{Mon DD} — {rounded temp}°{units} ({+/-N}° vs. 30-yr avg)"` — reuses `formatDelta`'s existing sign convention from `anomaly.ts` (Unicode minus, explicit `+`) rather than inventing a new delta format |
