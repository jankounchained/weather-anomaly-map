### New tokens — Anomaly Color System (DESIGN-02, DESIGN-04, D-02, D-06)

These are **computed/continuous**, not fixed-frequency 60/30/10 colors — documented separately because they don't fit that static model:

| Token | RGB | Hex | Role |
|-------|-----|-----|------|
| `--color-anomaly-cold` | `30, 58, 138` | `#1e3a8a` | Anchor at z ≤ −3 (coldest) |
| `--color-anomaly-normal` | `87, 83, 78` | `#57534e` | Anchor at z = 0 (near-normal) |
| `--color-anomaly-hot` | `154, 52, 18` | `#9a3412` | Anchor at z ≥ +3 (hottest) |
| `--color-atmosphere-night-wash` | `10, 8, 20` @ 40% | `rgba(10,8,20,0.4)` | Overlay wash applied atop the anomaly gradient when the pin's local time is night (D-03) |

**Pure mapping function contract (mirrors `anomaly.ts`'s pure-function-then-unit-test pattern — put this in `src/anomaly/` alongside `computeAnomaly`, not inline in a component):**

```
anomalyColor(zScore: number | null): string  // returns "#rrggbb"

1. Treat null zScore as z = 0 (same fallback precedent as classifyVerdict's
   `zScore ?? 0`, Pitfall 2) — resolves to --color-anomaly-normal exactly.
2. Clamp z to [-3, 3].
3. Two-segment piecewise-linear RGB interpolation (NOT a single cold→hot
   lerp — that would mud through gray/brown at the midpoint and would not
   pass exactly through the chosen "normal" anchor at z=0):
   - z <= 0: t = (z + 3) / 3;  color = lerp(COLD, NORMAL, t)
   - z >  0: t = z / 3;        color = lerp(NORMAL, HOT, t)
   - lerp per channel: round(c1 + (c2 - c1) * t), clamped to [0,255]
4. Output "#rrggbb".
```

Unit-test anchor points: z=-3 → exactly `#1e3a8a`; z=0 (and z=null) → exactly `#57534e`; z=+3 → exactly `#9a3412`; z=-1.5 and z=+1.5 → segment midpoints (spot-check the lerp math).

**Day/night is a separate, independent axis (D-03)** — it modulates the *backdrop gradient's* presentation only, never the hero text color (hero text always shows the pure `anomalyColor()` value for legibility consistency regardless of time of day):
```
isDaytime(localHour: number): boolean  =>  localHour >= 6 && localHour < 20
```
Night: layer `--color-atmosphere-night-wash` over the backdrop gradient (e.g. a second `background-image` layer or `::before` overlay) — darkens/desaturates the ambient mood without touching the anomaly hue's identity. Day: no wash, full anomaly-hue gradient at normal lightness.

**Required data-signal addition (verified, not assumed — CONTEXT.md D-03 flagged this for confirmation):** `useCurrentWeather`'s public `localDate` field is date-only (`client.ts`'s `localDateFrom` splits `current.time` on `"T"` and keeps only the date portion) — **the hour is currently discarded**, even though Open-Meteo's raw `current.time` ("YYYY-MM-DDTHH:mm", already pin-local per `timezone=auto`) contains it. Planning must add a sibling pure function `localHourFrom(currentTime: string): number` in `client.ts` and a new `localHour: number | null` field on `UseCurrentWeatherResult`, populated identically to `localDate` in `useCurrentWeather.ts`. **Zero new network calls** — this parses data already present in the existing forecast response, so it stays within "no new Open-Meteo calls."

## Layout & Composition (Glass System)

Resolves D-05's "composition is handed to the UI-SPEC" using the CONTEXT.md-recommended default, made concrete and buildable:

**Where the backdrop lives:** the docked `LocationPanel` (`<aside>`, currently `bg-secondary`, fixed 760px, beside the map — never over it) becomes the **static anomaly-gradient backdrop container**. The live map region (`flex-auto` on the left) is untouched — no gradient, no glass, no blur, exactly as Phase 4 left it. This satisfies PERF-01 **by construction**: nothing with `backdrop-blur` ever sits over `.leaflet-container`.

**No-pin / no-data state:** before a pin exists, or while the combined current+baseline fetch is still loading, there is no anomaly value to color by. The panel background stays the flat `--color-secondary` fallback (no gradient) rather than showing a placeholder-colored gradient that would misleadingly imply "this is the anomaly" before one is computed. Once `hasSelection` is true and the fetches are in flight, use `anomalyColor(null)` (i.e., the normal anchor) as the gradient base — this is a legitimate "neutral, waiting" mood, not a misleading claim of a specific reading.

**Three glass cards inside the panel** — one per existing content region, applied to the **root container of every state branch** (empty / loading / error / resolved), not just the happy path, so the glass treatment never flickers/resets between states:
1. `LocationDisplay` root
2. `AnomalyCard` root (the hero — gets stronger surface opacity, see below)
3. `TrendRow` root (including each `TrendDayChart`'s "not enough data" placeholder tile)

Shared glass card treatment:
- Background: `rgba(255,255,255,0.62)` (new token `--color-glass-surface`)
- Border: `1px solid rgba(255,255,255,0.45)` (new token `--color-glass-border`)
- Corner radius: `16px` for the three panel-width cards (new token `--radius-glass-lg`); the small 88px trend-tile "not enough data" box keeps its existing `8px` radius (new token `--radius-glass-sm`, same value as today — no change, just formalized)
- Blur: real `backdrop-blur` at Tailwind's `lg` (16px) on the three panel-width cards only. **Do not** add a second nested blur on the small trend tiles — they already sit inside the already-blurred `TrendRow` card; stacking blur on blur wastes paint cost for no visible gain
- Shadow: `0 8px 30px rgba(15,8,30,0.14)` (new token `--shadow-glass`)
- Padding: reuse existing spacing scale — `px-md py-md` (16px) for the two smaller cards, hero card gets `p-lg` (24px)

**Hero elevation (DESIGN-04 — the hero must read as the unmistakable focal point):** the `AnomalyCard` glass card gets a **more opaque** surface than its siblings — `rgba(255,255,255,0.72)` — plus the standard shadow. This is the mechanism for "strengthened hierarchy": highest surface opacity (best legibility guarantee, satisfying D-06's hard contrast requirement), the color-coded delta at 700-weight/48px, and (by virtue of being first in reading order under the location name) top visual position. The z-score chip stays visually subordinate: keep its existing neutral `bg-secondary`/`text-muted` pill styling unchanged — it must **not** take anomaly-hue coloring; that would compete with the hero delta for "which number is the answer."

**Bridging the computed color into CSS (required, scoped exception to the "no inline style" convention):** `anomalyColor()` produces a per-render, data-driven hex value — this cannot be expressed as a static Tailwind class or a static `@theme` token, the same way the existing static `var(--color-chart-*)` tokens can. The correct, minimal bridge: set a single CSS custom property via inline `style` on the `LocationPanel` root — `style={{ '--anomaly-color': anomalyColor(zScore) } as React.CSSProperties}` — and have all downstream CSS (gradient stops, hero text color) read `var(--anomaly-color)`. This is explicitly **not** a reopening of "no inline style" broadly: it is one narrowly-scoped bridge point for one data-driven value, directly analogous to how recharts already bridges via inline `var(--token)` SVG props. Do not introduce additional inline styles beyond this single custom-property assignment.

**Recommended gradient (implementation guidance, not a hard mandate on exact syntax):**
```css
@property --anomaly-color {
  syntax: '<color>';
  inherits: true;
  initial-value: #57534e;
}
/* registering the custom property makes it natively interpolable for the
   one-time entrance transition below — see Motion & Performance Contract */

.panel-backdrop {
  background: linear-gradient(155deg,
    color-mix(in srgb, var(--anomaly-color) 80%, white 20%) 0%,
    var(--anomaly-color) 55%,
    color-mix(in srgb, var(--anomaly-color) 75%, black 25%) 100%);
}
.panel-backdrop.is-night::before {
  content: '';
  position: absolute;
  inset: 0;
  background: var(--color-atmosphere-night-wash);
  pointer-events: none;
}
```

## Motion & Performance Contract (PERF-01, PERF-02, D-10)

- **PERF-01 compliance:** real `backdrop-blur` exists **only** on the three glass cards inside the docked `LocationPanel`, which sits beside — never over — `.leaflet-container`. No component floats over the live map. This is a structural guarantee from the composition above, not a runtime check to add.
- **PERF-02 / motion:** a **one-time entrance transition** when the backdrop or hero recolors on a **new pin** (or on the loading→resolved anomaly transition). Use Tailwind's `motion-safe:` variant directly — it already wraps `prefers-reduced-motion: no-preference` natively, so no custom media query is needed:
  - `motion-safe:transition-colors motion-safe:duration-300 motion-safe:ease-out` on the panel backdrop element and on the hero delta text element.
  - Registering `--anomaly-color` via `@property` (above) is what makes this transition **actually interpolate the color** smoothly rather than snapping — without it, most browsers do not animate an un-registered custom-property-driven gradient.
- **Hard rule:** no `requestAnimationFrame`, no canvas, no JS-driven interval/timeout animation loop anywhere. The entrance transition is pure CSS, triggered implicitly by React re-rendering with a new `--anomaly-color` value — never orchestrated by JS timing code.
- With `prefers-reduced-motion: reduce`, `motion-safe:` utilities emit no transition at all — the backdrop/hero snap instantly to the new color. Verify this is the actual behavior (no fallback JS-driven fade path exists to accidentally bypass the media query).
