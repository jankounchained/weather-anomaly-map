## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public internet → Cloudflare Pages static site | Anyone (unauthenticated) can reach the app; there is no auth boundary by design (PLAT-01) |
| Shared URL → app on load | User-supplied query params on the live site are read into state — same input-validation boundary as Plan 01, now exposed publicly |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|

Verified end-to-end at the checkpoint (step 5). |
| T-01-04 | Information disclosure | Deployed static bundle / hosting config | low | mitigate | No secrets, API keys, or env vars exist in the build (both APIs keyless, PLAT-02); Task 1 asserts no env vars are configured, so nothing sensitive can leak via the bundle or the Pages project settings. |
| T-01-SC | Tampering | Deployed dependency bundle (supply chain) | high | mitigate | The production bundle is built only from the RESEARCH.md-audited, exact-pinned dependencies (all Approved) carried forward from Plan 01; no new packages are introduced in this plan, so no additional legitimacy review is required. |
</threat_model>
