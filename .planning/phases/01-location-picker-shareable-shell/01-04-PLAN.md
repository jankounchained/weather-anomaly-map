<objective>
Close the Phase 1 verification BLOCKER: the shareable-URL round-trip is broken for a common Leaflet interaction. `setLocation` — the write path invoked on every map click and drag — rounds lat/lng but never clamps latitude or wraps longitude, so a click after world-copy panning writes an out-of-range value (e.g. `?lng=380`) that the read path then silently rejects, substituting the Czech Republic default instead of the shared location (CR-01). This plan makes the write path enforce the same [-90,90]/[-180,180] invariant the read path already enforces, wrapping (not clamping) longitude so distinct points stay distinct, and adds the renderHook test coverage whose absence let this ship (IN-03). It also closes the same-root-cause companion (WR-01: `hasUrlSelection` checks key presence not validity → phantom pin + wasted geocode) and folds in three co-located hardening fixes (WR-02 side-effect-in-updater, WR-03 untyped geocode response, IN-01 lenient numeric parse, IN-02 dead CSS token annotation).

Purpose: The URL query string is this app's only persistence layer (no backend). "Anyone with the link sees the same location" is the phase's core promise — it currently fails for a non-exotic interaction. Re-verification will re-check the FAILED truth #7; this plan is what makes it pass.

Output: A location write path that is provably in-range for any input, a validated hasUrlSelection gate, and direct hook-level test coverage of the previously-untested clamping path.
</objective>

## Artifacts This Plan Produces

- `wrapLng(lng)` — `src/lib/coords.ts`. Pure cyclic-longitude normalizer: `((((lng + 180) % 360) + 360) % 360) - 180`. Idempotent for in-range values.
- Rewritten `setLocation` — `src/map/useSelectedLocation.ts`. Applies `clampLat(round4(lat))` and `round4(wrapLng(lng))` before both the URL write and the state update; performs the `writeLocationToUrl` side effect OUTSIDE the state updater (WR-02).

- `isValidUrlSelection(search?)` — `src/map/useSelectedLocation.ts`. New exported boolean helper reusing `parseGuarded` so App and the hook validate identically (WR-01).
