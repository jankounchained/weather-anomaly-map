<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → BigDataCloud API (`api.bigdatacloud.net`) | Untrusted third-party JSON (place-name strings) crosses back into the app and is rendered in the panel |
| Geocode response → DOM | Response text is displayed to the user; a malicious/tampered response must not become executable markup |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-02 | Tampering / Information disclosure | `LocationDisplay` rendering the geocode place-name string | medium | mitigate | Render the place name and coordinates as ordinary React JSX text nodes only (React auto-escapes text content); no raw-HTML rendering sink is used anywhere on this path, so a tampered response cannot inject executable markup. |
| T-01-03 | Denial of Service | `reverseGeocode` fetch to BigDataCloud | low | mitigate | AbortController ~3s timeout plus coordinate fallback (D-02): any outage, latency, non-2xx (incl. 402 fair-use ban), or malicious response resolves to the coordinate display within the deadline — never an indefinite spinner, never code execution. Response is read-only display data only. |
</threat_model>
