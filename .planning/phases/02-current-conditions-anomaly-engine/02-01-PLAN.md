## Phase Goal

**As a** visitor checking a location, **I want to** see today's temperature and how it compares to the 30-year norm for this calendar day, **so that** I can tell at a glance whether today is unusually warm or cold.

> This plan (Plan 01) delivers the first vertical slice of that goal: drop a pin → see today's current temperature. Plan 02 adds the anomaly (delta, z-score, verdict). Plan 03 is the human verification checkpoint for the full card.

<objective>
Deliver the current-temperature vertical slice (CURR-01): a user who drops a pin sees today's actual temperature for that location, fetched live from Open-Meteo's forecast endpoint through a hook that mirrors the existing `useReverseGeocode` idle/loading/resolved pattern.

Purpose: This is the thinnest end-to-end weather slice — it stands up the `src/weather/` domain folder (types, fetch client, hook) and the `AnomalyCard` shell that Plan 02 will extend with the anomaly numbers. After this plan a real user can do something they could not before: see the live temperature for any pinned location.

Output: `src/weather/{types,client,client.test,useCurrentWeather}.ts` and `src/app/AnomalyCard.tsx`, wired into `App.tsx` and `LocationPanel.tsx` with supporting styles in `App.css`.
</objective>
