## Spacing Scale

Declared values (must be multiples of 4) — **unchanged from `src/index.css`, already declared and in production use**:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding (existing panel `p-lg`/`py-lg px-md`) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **InfoTooltip trigger button: 20px (`h-5 w-5`)**, matching the existing "i" data-quality button in `AnomalyCard.tsx`. Below the general 44px touch-target guidance — accepted because v1.2 is a desktop-focused reflow (PLAT-03 mobile layout is deferred); the trigger must still carry a visible `:focus-visible` ring so it remains keyboard-discoverable at this size.
- No other exceptions.

## Typography

Roles unchanged from `src/index.css` (already declared, exactly 4 sizes / 2 weights):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 400 (headline eyebrows use 600 via `font-semibold` utility, not a new weight token) | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

**Delta dominance rule (LAYOUT-03, locked):** the Delta panel's Δ-glyph number renders at `calc(var(--text-display) * 1.7)` = **47.6px**, exactly as it already does in `AnomalyCard.tsx` today. This is *not* a 5th independent size token — it is Display scaled by a fixed 1.7× multiplier, which sits inside the required 1.5–2× dominance band versus the Current Conditions panel's plain Display-sized temperature (28px). When splitting the hero into two panels, preserve this multiplier verbatim; do not let either panel's number drift outside the 1.5–2× ratio.

## Color

Unchanged from `src/index.css` (already declared, 60/30/10 split in production):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (`--color-dominant`) | Map background, page background |
| Secondary (30%) | `#f4f5f7` (`--color-secondary`) | Docked panel background (empty state), z-score chip pill background |
| Accent (10%) | `#2563eb` (`--color-accent`) | InfoTooltip trigger border/glyph color, InfoTooltip trigger `:focus-visible` ring, loading-spinner active arc, chart mean reference line |
| Destructive | `#dc2626` (`--color-destructive`) | Error-state copy only ("Couldn't compute an anomaly here.", "Temperature unavailable for this location.") — no destructive actions exist in this phase |

Accent reserved for: **InfoTooltip trigger glyph/border, InfoTooltip trigger focus ring, the loading-spinner's spinning arc, and the trend chart's mean reference line.** Never applied to panel headlines, body copy, or the Delta number itself.

**Dynamic anomaly-color token (carried over from Phase 5, not part of the 60/30/10 split):** `--anomaly-color` is a registered `@property` continuous z-score→color lerp, currently bridged onto `LocationPanel`'s `.panel-backdrop` and consumed by the Delta glyph via `style={{ color: 'var(--anomaly-color)' }}`. **This phase must keep the Delta panel's number reading `var(--anomaly-color)`** regardless of which DOM level now hosts the CSS custom property — the color-driven focal treatment (STATE.md v1.1/Phase 5 decision) is a hard carry-over, not optional. Do not add `backdrop-blur` to any element that sits over the live Leaflet map — the disciplined-glass performance policy (no blur over the map) applies unchanged; the docked panel sits beside the map, not over it, so its existing `backdrop-blur-lg` glass cards are unaffected and stay as-is.

## Component Inventory (new primitives this phase)

Per ROADMAP's research flag, Phase 6 establishes three shared primitives reused verbatim by Phases 7 and 8. Downstream phases must import these, not re-derive the glass/eyebrow/tooltip patterns inline.

| Primitive | Responsibility | Style source |
|-----------|----------------|---------------|
| `PanelShell` | Wraps the repeated glass-card class string (`bg-glass-surface border border-glass-border rounded-glass-lg shadow-glass backdrop-blur-lg` + padding) currently duplicated across `LocationDisplay`, `AnomalyCard`, `TrendRow`. Accepts a `as` prop for the semantic element (`aside`/`section`/`div`) and children. | Existing glass tokens in `src/index.css` — no new tokens |
| `PanelHeadline` | Renders the eyebrow label style already proven in `TrendRow.tsx` (`text-label leading-[1.5] font-semibold text-muted uppercase tracking-[0.05em]`). Takes headline text as children. Used by all four panels (Location, Current Conditions, Delta, History). | `TrendRow.tsx` line 38 (verbatim reuse, do not restyle) |
| `InfoTooltip` | Accessible info affordance replacing the ad hoc `title`-attribute "i" button in `AnomalyCard.tsx`. Trigger keeps the existing visual (`h-5 w-5 rounded-full border border-accent text-accent`, `i` glyph) but gains a real disclosure/popover behavior (see Interaction Contract below). | Trigger visual from `AnomalyCard.tsx` lines 78–85; popover body uses `PanelShell`'s glass surface at a smaller `max-w-[240px]` |

**InfoTooltip interaction contract (EXPLAIN-02 — must satisfy WCAG 1.4.13 hoverable/dismissible/persistent):**
- Trigger is a real `<button type="button">` with `aria-expanded` and `aria-controls` pointing at the popover id.
- Opens on: click/Enter/Space (persists until dismissed) **and** on mouse hover/keyboard focus (progressive enhancement) — hover-opened content must stay open while the pointer moves onto the popover itself.
- Closes on: `Escape` (returns focus to the trigger), click outside, or blur when not hover-pinned.
- Popover role: `role="dialog"` with a visually-hidden or `aria-label` naming the panel it explains (e.g. `aria-label="About the delta and z-score"`) — not `role="tooltip"`, because the content is multi-sentence prose, which exceeds what a native tooltip role should carry.
- Popover text wraps at `max-w-[240px]` — never causes horizontal overflow of the 760px docked panel column.
