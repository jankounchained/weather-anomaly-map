<objective>
Fix UAT gap G-06-11: both panels' InfoTooltip popover is unreadable when opened — the Current Conditions popover paints behind the panels, and the Delta popover renders clipped off the right edge of the frame.

Root cause (from 06-UAT.md): the popover is an in-flow `absolute z-10` descendant of PanelShell, whose `backdrop-blur-lg` (backdrop-filter) creates a new stacking context per panel. z-index cannot escape an ancestor backdrop-filter stacking context, so the earlier-sibling Current Conditions popover is painted beneath the later-sibling Delta panel. Separately, the popover is left-anchored to its trigger with no viewport-edge flip while App's root wrapper is `overflow-hidden`, so the right-edge Delta popover is clipped off-frame.

Fix: lift the popover into a React portal on document.body with `position: fixed` computed from the trigger's `getBoundingClientRect()`, add viewport-edge-aware flip/shift so the right-edge popover stays in frame, and add a popoverRef so the existing WCAG 1.4.13 hover/focus/persist/outside-click contract survives the popover leaving the trigger's DOM subtree.

Purpose: EXPLAIN-02's info affordance is only useful if its explanation is actually readable; today it is not, on either panel.
Output: A pure edge-aware positioning helper (unit-tested), a portaled/fixed-position InfoTooltip, and extended tests proving portal placement, edge-flip, and the preserved interaction contract.
</objective>

Input shape: `{ triggerRect: { top, bottom, left, right, width, height }, popoverWidth, popoverHeight, viewportWidth, viewportHeight, gap, margin }`. Export the input/output TypeScript types too.

    Rules (a pure function of its inputs — no DOM access, no globals):
    - Default vertical: place BELOW the trigger — top = triggerRect.bottom + gap.
    - Default horizontal: left-align to the trigger — left = triggerRect.left.
    - Right-edge shift (the Delta fix): if triggerRect.left + popoverWidth > viewportWidth - margin, set left = viewportWidth - margin - popoverWidth so the popover's right edge sits at viewportWidth - margin.
    - Left clamp: left is never less than margin.
    - Bottom-edge flip: if triggerRect.bottom + gap + popoverHeight > viewportHeight - margin, flip ABOVE the trigger — top = triggerRect.top - gap - popoverHeight.
    - Top clamp: top is never less than margin.

    Tests (write first, watch them fail, then implement):
    - Normal left-side trigger: triggerRect {left:100,right:140,top:100,bottom:120,width:40,height:20}, popoverWidth 240, popoverHeight 100, viewport 1000x800, gap 8, margin 8 -> { top: 128, left: 100 }.

    - Left underflow clamps: triggerRect.left -20 -> left equals margin (8).

Attach `onMouseEnter` on the container as today (opens hover-pinned); the popover wrapper only needs the shared leave handler.
